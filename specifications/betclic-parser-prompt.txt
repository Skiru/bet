Build `scripts/parse_betclic_html.py` — Betclic HTML parser for all 5 sports

Build a parser that extracts matches, odds, leagues, and market data from cached Betclic HTML pages for all 5 core sports (football, basketball, tennis, volleyball, hockey).

Context files to read first:
- `specifications/betclic-dom-selectors.md` — complete DOM selector map with URL patterns, listing page selectors, match detail selectors, odds button structure, market tabs per sport, and the full standard market catalog (Polish → English) for all 5 sports
- `betting/data/html_cache/` — cached HTML files (4 listing pages + 3 match detail pages from 2026-05-13)
- `scripts/stealth_utils.py` — stealth Playwright utilities (for fetching fresh pages when needed)
- `scripts/daily_odds_warmup.py` — existing S0.1 warmup that caches listing pages (the parser's input source)

What the parser should do:
1. Parse listing pages (2026-MM-DD_betclic_*.html) — extract all events: home/away teams, league, kickoff time, live status, match URL, listing odds (1X2 or ML), market count
2. Parse match detail pages (2026-MM-DD_betclic_match_*.html) — extract all markets from the active tab: market name, selections with labels and odds values, over/under lines
3. Output structured JSON per sport + combined summary, suitable for pipeline consumption (DB insertion or shortlist matching)
4. Handle Polish locale — comma decimals ("8,25" → 8.25), Polish market names mapped to English pipeline equivalents
5. Match ID extraction from URLs — the mNNNNNNNNNNNNNNN pattern for deduplication

Key DOM facts (from our analysis):
- Listing: `sports-events-event-card` → `a.cardEvent[href]`, `[data-qa="contestant-1-label"]`, odds in `bcdk-bet-button-label.btn_label`
- Match detail: `sports-markets-single-market` → `.marketBox_headTitle` for market name, `.marketBox_label` for selections, `.marketBox_lineSelection` for odds
- Only the active tab renders odds — default is "Top" which covers the key markets
- Football has 8 tabs: MyCombi, Top, Wynik, Strzelcy (scorers), Gole, Metoda gola (goal method), Wynik/Handicap, Statystyki (corners/cards/shots/fouls)
- The Statystyki tab is critical — it has our statistical markets (corners O/U, cards O/U, shots on target, fouls)
- Team names are split: span.ellipsis + span.clip
- Max ~20 events per listing page — for full coverage we need per-league pages or scrolling

Integration target: The parser output should be usable by the existing pipeline — either stored in betting.db (fixtures + odds tables) or as JSON consumed by build_shortlist.py and generate_market_matrix.py.
