# User Betting Preferences

## Core Philosophy
- Statistical markets first: corners, cards, fouls, shots, games, sets, frames.
- Goals/BTTS/ML are fallback markets only when no statistical market available.
- Prefer multi-sport coverage and less efficient leagues when data supports it.
- Keep picks conditional; Betclic odds verified by user in app.
- Use full team/player names and clear Polish market descriptions.

## Practical Notes
- Working bankroll is small; leave budget unused when board is weak.
- Betclic Statystyki tab is limited and not available for every league.
- Do NOT scrape Betclic for odds; treat all prices as reference only.
- Deep source fusion: combine tipster opinions + DB stats + web search.
- Never give up on data: DB → scrapers → web search → NEVER "no data".
